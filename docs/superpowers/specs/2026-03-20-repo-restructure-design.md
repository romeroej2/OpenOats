# Repo Restructure: Break from Fork

**Date:** 2026-03-20

## Context

OpenCassava was forked from `yazinsai/OpenOats`. Now that we're going our own direction, we no longer need fork compatibility. This cleanup flattens the repo structure, removes fork artifacts, and removes dead git state.

## Target Structure

```
repo-root/
├── opencassava/          (was: OpenCassava/OpenCassavaTauri/)
│   ├── src/              React/TypeScript UI
│   ├── src-tauri/        Tauri + Rust bridge
│   ├── package.json
│   └── ...
├── crates/
│   └── opencassava-core/
├── Cargo.toml            (was: OpenCassava/Cargo.toml)
├── Cargo.lock            (was: OpenCassava/Cargo.lock)
├── package.json          (was: OpenCassava/package.json)
├── package-lock.json
├── README.md
├── LICENSE
├── assets/
└── docs/
```

## Changes

### Prerequisites (must happen first)
1. **Rename the GitHub repo** from `OpenOats` to `OpenCassava` on GitHub before updating the git remote URL — otherwise the push will fail.

### Git moves (history-preserving, use `git mv`)
- `OpenCassava/OpenCassavaTauri/` → `opencassava/`
- `OpenCassava/crates/` → `crates/`
- `OpenCassava/Cargo.toml` → `Cargo.toml`
- `OpenCassava/Cargo.lock` → `Cargo.lock`
- `OpenCassava/package.json` → `package.json`
- `OpenCassava/package-lock.json` → `package-lock.json`

### Deletions (use `git rm`)
- `OpenCassava/` — wrapper directory, empty after moves above
- `output/` — empty directory (untracked, just `rm -rf`)
- `scripts/` — empty directory (untracked, just `rm -rf`)
- `opencassava/src-tauri/err.txt` — build artifact (tracked, use `git rm`)
- `opencassava/src-tauri/OpenCassavaWindows.dll` — compiled binary artifact committed to git; should not be tracked (use `git rm`)
- `.claude/worktrees/relaxed-antonelli` — dead worktree recorded as a **gitlink** (mode `160000`); must use `git rm -r .claude/worktrees/` (filesystem deletion alone will leave a dangling entry in the git index)

### Git remotes
- Remove `upstream` remote (`yazinsai/OpenOats.git`): `git remote remove upstream`
- Update `origin` to `romeroej2/OpenCassava.git`: `git remote set-url origin https://github.com/romeroej2/OpenCassava.git` (do this after the GitHub rename)

### Path reference updates

**`Cargo.toml` (workspace root, moved from `OpenCassava/Cargo.toml`):**
- Update workspace members from `crates/opencassava-core` and `OpenCassavaTauri/src-tauri` to `crates/opencassava-core` and `opencassava/src-tauri`
- The `[patch.crates-io]` entry `whisper-rs = { path = ".build/whisper-rs" }` resolves relative to the workspace root. After the move, workspace root is the repo root, so `.build/whisper-rs` must exist at `<repo-root>/.build/whisper-rs`. This path is untracked (gitignored) and must be regenerated by running the prepare script before the first build. Document this in the build instructions.

**`opencassava/src-tauri/Cargo.toml`:**
- Contains `opencassava-core = { path = "../../crates/opencassava-core" }`. After the move, `opencassava/src-tauri/` is two levels from repo root, so `../../crates/` resolves correctly. Verify this path is still valid post-move before committing.

**`.gitignore`:**
Update all `OpenCassava/`-prefixed entries and remove the stale `/OpenOats/.build` entry:
- `OpenCassava/.build/` → `.build/`
- `OpenCassava/node_modules/` → `node_modules/` and `opencassava/node_modules/`
- `OpenCassava/*.log` → `*.log` (or path-specific)
- `OpenCassava/target/` → `target/`
- `OpenCassava/OpenCassavaTauri/.npm-cache/` → `opencassava/.npm-cache/`
- `OpenCassava/Cargo.lock` → remove (Cargo.lock now tracked at root)
- `OpenCassava/package-lock.json` → remove (now tracked at root)
- `/OpenOats/.build` → remove entirely (stale entry from before rename)
- Add `.claude/worktrees/` to prevent future dead worktrees from being tracked

**`.github/workflows/windows-release.yml`:**
- Update any path references from `OpenCassavaTauri/` to `opencassava/`
- Update the rust-cache workspace entry (line ~38): `OpenCassava -> target` → `. -> target` (the workspace root is now the repo root)

**`README.md`:**
- Update all `OpenCassavaTauri` references in the "Build from source" section (the `cd` line, the `node_modules` prose line, and the installer output path line)
- Replace the full architecture diagram (the tree block) — it references `OpenCassava/` prefix, `Sources/`, and `Package.swift` which no longer exist

**`docs/lm-studio-setup.md`:**
- Update all `OpenCassavaTauri` references — both the `cd OpenCassava\OpenCassavaTauri` line and the `OpenCassavaTauri\node_modules` prose line

### Post-restructure build step
After the restructure, `.build/` no longer exists at the new workspace root. Before the first build, run:
```
cd opencassava
npm run tauri -- build
```
or run the `prepare-whisper.ps1` script directly. This regenerates `.build/whisper-rs` at the repo root where the workspace `Cargo.toml` now expects it.

## Approach

1. Rename GitHub repo to `OpenCassava` (prerequisite)
2. Use `git mv` for all file moves to preserve history
3. Use `git rm` for deletions of tracked files/gitlinks
4. Fix all path references in the files listed above
5. Commit everything as a single "repo restructure" commit
6. Update git remotes

No force-push required.

## Out of Scope

- No code changes
- No dependency updates
- No feature work
- `prepare-whisper.ps1` still uses `.openoats-patched-commit` as its cache stamp filename — stale from before the rename. Track as follow-up technical debt.
